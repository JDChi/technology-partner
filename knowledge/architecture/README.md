# Architecture Notes

Use this directory for durable understanding of repositories, systems, and component relationships.

Each architecture note should try to answer:
- what the system or subsystem does
- how the main pieces interact
- where the important boundaries are
- what constraints shape the design
- what risks or scaling concerns are visible

Recommended file naming:
- `repo-name-overview.md`
- `service-name-architecture.md`
- `feature-name-flow.md`
