# Decision Records

Use this directory for durable technical decisions and their rationale.

A good decision record should include:
- the context
- the decision
- alternatives considered
- tradeoffs
- applicable scenarios
- what would cause this decision to be revisited

Recommended file naming:
- `YYYY-MM-DD-topic.md`
- `stack-selection-backend.md`
- `data-sync-strategy.md`
